## 中间件、守卫、拦截器、过滤器 对比

### 加载顺序（请求生命周期）

```
请求进入
   │
   ▼
┌─────────────────────────┐
│      Middleware         │  ← 第一层：全局中间件 → 模块中间件
│  （请求/响应拦截）        │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│        Guard            │  ← 第二层：路由守卫（权限校验）
│   （路由守卫）           │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│     Interceptor         │  ← 第三层：拦截器（请求/响应转换）
│  （请求/响应拦截）        │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│        Pipe             │  ← 第四层：管道（参数转换/验证）
│   （参数处理）           │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│      Controller         │
│    （控制器）            │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│      Service            │
│    （服务层）            │
└─────────┬───────────────┘
          │
          ▼
     业务处理
          │
          ▼
┌─────────────────────────┐
│      Interceptor        │  ← 响应拦截（后置处理）
│   （响应转换/后置处理）   │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│       Filter           │  ← 最后层：异常过滤器（异常处理）
│   （异常处理）           │
└─────────┬───────────────┘
          │
          ▼
      响应返回
```

***

### 各组件对比

| 特性           | 中间件      | 守卫    | 拦截器   | 过滤器   |
| :----------- | :------- | :---- | :---- | :---- |
| **执行时机**     | 请求进入时    | 路由匹配后 | 请求前后  | 只在异常时 |
| **应用范围**     | 全局/模块/路由 | 路由    | 全局/路由 | 全局/路由 |
| **能访问请求**    | ✅        | ✅     | ✅     | ✅     |
| **能访问响应**    | ✅        | ❌     | ✅     | ❌     |
| **能修改请求/响应** | ✅        | ❌     | ✅     | ❌     |
| **能抛出异常**    | ✅        | ✅     | ✅     | ✅     |
| **依赖注入**     | ✅        | ✅     | ✅     | ✅     |

***

### 1. 中间件 (Middleware)

**定义**：连接两个不同部分的软件

**特点**：

- 最先执行，在路由匹配之前
- 可以访问 `req`、`res`、`next`
- 可以修改请求和响应
- 可以结束请求周期

**应用场景**：

- CORS 跨域处理
- 请求日志记录
- 请求体解析（body-parser）
- 静态文件服务

```typescript
@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: NextFunction) {
    console.log(`[${new Date()}] ${req.method} ${req.url}`);
    next(); // 必须调用
  }
}
```

***

### 2. 守卫 (Guard)

**定义**：决定请求是否被处理

**特点**：

- 在路由匹配之后、拦截器之前执行
- 只能返回 `true`/`false`
- 不能修改请求/响应
- 常用于权限校验

**应用场景**：

- 用户登录验证
- 角色权限校验
- API Key 验证
- 路由访问控制

```typescript
@Injectable()
export class AuthGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest();
    const token = request.headers.authorization;
    
    if (!token) {
      throw new UnauthorizedException();
    }
    return true;
  }
}
```

***

### 3. 拦截器 (Interceptor)

**定义**：在请求和响应阶段进行转换

**特点**：

- 使用 RxJS 的 Observable
- 可以在请求前和响应后执行逻辑
- 可以转换响应数据
- 支持响应包装（REST → 统一格式）

**应用场景**：

- 请求日志记录（计时）
- 响应数据转换（null → ''）
- 统一响应格式 `{ code, data, message }`
- 缓存处理

```typescript
@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const start = Date.now();
    
    return next.handle().pipe(
      tap(() => console.log(`耗时: ${Date.now() - start}ms`)),
      map(data => ({ data, code: 0 }))
    );
  }
}
```

***

### 4. 过滤器 (Exception Filter)

**定义**：专门处理异常

**特点**：

- 只在抛出异常时执行
- 可以捕获任意类型的异常
- 可以自定义响应格式
- 在请求生命周期最后执行

**应用场景**：

- 统一错误响应格式
- 日志记录异常
- 自定义业务异常处理
- 异常监控上报

```typescript
@Catch()
export class GlobalExceptionFilter implements ExceptionFilter {
  catch(exception: unknown, host: ArgumentsHost) {
    const response = host.switchToHttp().getResponse();
    const status = exception instanceof HttpException 
      ? exception.getStatus() 
      : 500;
    
    response.status(status).json({
      statusCode: status,
      message: exception.message,
      timestamp: new Date().toISOString(),
    });
  }
}
```

***

### 总结对比

| 组件      | 职责      | 执行时机    | 典型场景             |
| :------ | :------ | :------ | :--------------- |
| **中间件** | 通用请求处理  | 最先      | CORS、日志、解析       |
| **守卫**  | 权限决策    | 路由后、拦截前 | 登录校验、角色校验        |
| **拦截器** | 请求/响应转换 | 路由前后    | 日志计时、数据转换、统一响应格式 |
| **过滤器** | 异常处理    | 只在异常时   | 错误响应格式化、异常日志     |

### 选择建议

- **需要处理请求体/跨域** → 中间件
- **需要权限校验** → 守卫
- **需要转换响应/记录日志** → 拦截器
- **需要处理错误** → 过滤器
